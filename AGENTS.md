# AGENTS.md

本文件用于说明本项目的基本信息、目录结构、开发约定与工作注意事项，方便后续的自动化代理或协作者快速接手。

## 1. 项目概览

- 项目名称：`xiang-huo-sichuan`
- 项目类型：单页餐厅概念站点
- 主题定位：以“巷火川菜馆”为核心视觉与文案主题，展示川菜餐厅的菜单、品牌故事、食客评价和订位表单
- 数据属性：当前页面中的品牌、地址、电话和订位流程均为概念演示用途，非真实业务系统

## 2. 技术栈

- 前端框架：React
- 构建工具：Vite
- 语言：TypeScript
- 图标库：`lucide-react`
- 样式：原生 CSS，集中在 `src/styles.css`

## 3. 常用脚本

以下命令定义在 `package.json` 中：

- `pnpm dev`：启动本地开发服务
- `pnpm build`：执行类型检查并输出生产构建
- `pnpm preview`：本地预览构建产物
- `pnpm typecheck`：仅执行 TypeScript 类型检查

## 4. 目录结构

- `src/main.tsx`：应用入口，挂载 React 根节点并引入全局样式
- `src/App.tsx`：页面主体，负责首屏、菜单、故事、评价、到店信息和订位弹窗
- `src/data.ts`：页面数据与 TypeScript 类型定义，包括菜品、评价、门店信息和表单结构
- `src/styles.css`：全部页面样式、响应式布局、动画与弹窗样式
- `public/images/`：页面静态图片资源
- `dist/`：构建输出目录

## 5. 页面内容说明

页面由以下几个区块组成：

1. 顶部导航与主视觉区
2. 菜单区：支持按分类筛选菜品
3. 品牌故事区：展示餐厅理念与食材说明
4. 食客评价区：展示评分与评论卡片
5. 到店信息区：展示地址、营业时间、电话和路线示意
6. 订位弹窗：前端模拟表单，提交后仅显示演示成功状态

## 6. 开发约定

- 优先使用中文补充页面文案、注释和说明，保持项目整体表达统一
- 菜单、评价、门店信息等静态内容优先维护在 `src/data.ts`
- 页面结构和交互逻辑优先维护在 `src/App.tsx`
- 样式尽量集中写在 `src/styles.css`，避免在组件中散落复杂样式逻辑
- 若需要新增图片资源，优先放入 `public/images/`

## 7. 交互与实现细节

- 订位表单仅做前端校验与演示，不会真实提交或保存数据
- 弹窗打开时会锁定页面滚动，按 `Esc` 可关闭
- 页面含有基于 `IntersectionObserver` 的滚动进入动画
- 菜品列表支持分类筛选，分类值由 `DishCategory` 类型统一约束

## 8. 维护时的注意事项

- 当前项目是概念展示站点，新增功能时请先判断是否需要接入后端
- 修改文案后，注意同步检查图片 `alt`、按钮标签和无障碍属性
- 修改菜品分类或字段时，需要同时更新：
  - `src/data.ts` 中的数据与类型
  - `src/App.tsx` 中的筛选逻辑和展示逻辑
  - 相关样式与响应式规则
- 生产构建依赖 TypeScript 严格检查，改动后建议至少运行 `pnpm typecheck`

## 9. 当前项目风格

- 整体视觉偏复古、浓烈、餐饮海报感
- 文案风格偏口语化、具画面感
- 适合保留较强的品牌氛围，不建议轻易改成极简科技风

## 10. 快速上手建议

如果你要继续修改这个项目，通常按下面的顺序最稳妥：

1. 先看 `src/data.ts`，确认内容数据结构
2. 再看 `src/App.tsx`，理解页面结构与交互
3. 最后改 `src/styles.css`，处理展示和响应式效果
